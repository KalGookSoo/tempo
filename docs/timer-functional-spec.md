# Tempo 타이머 기능명세서

## 목적

이 문서는 Rogue Echo Gym Timer Clock 매뉴얼의 타이머 기능을 참고하여 tempo MVP에 맞게 정리한 기능명세서다.
tempo는 React Native 기반 모바일 앱으로 구현하며, 별도 물리 리모컨이나 하드웨어 전자시계는 현재 범위에서 제외한다.

기존 하드웨어 타이머의 핵심 운동 타이머 기능은 tempo에서도 올바르게 동작해야 한다. 다만 모바일 앱에 맞지 않는 기능은 제거하거나 앱 UX에 맞게 재해석한다.

## 범위

### 포함

- 타이머: 카운트다운 / 카운트업 토글
- 스톱워치
- 인터벌 타이밍
- 기본 제공 인터벌 프리셋
- 사용자 프리셋 생성, 편집, 이름 지정, 삭제
- 알림 큐
- 사운드, 진동, 음원 파일, 직접 녹음 음성 큐
- 준비 카운트다운
- 화면 미러링을 고려한 실행 화면
- 모바일 제어 UI 매핑

### 제외

- 실시간 시계
- 물리 리모컨 제작
- 적외선 송수신
- 하드웨어 전자시계 연동
- 벽걸이 설치, 전원 어댑터, 배터리 등 하드웨어 설치 절차

## 제외 기능 판단

### 실시간 시계

실시간 시계는 기존 하드웨어 전자시계에서는 유용하지만, tempo MVP에서는 제외한다.

- tempo는 모바일 운동 타이머 앱이다.
- 모바일 기기에는 이미 시스템 시계가 있다.
- MVP의 핵심은 운동 타이머, 프리셋, 알림 큐다.
- 하드웨어 전자시계를 지원하는 단계가 오기 전까지 별도 실시간 시계 모드는 제공하지 않는다.

## 공통 개념

### 타이머 상태

타이머는 다음 상태를 가진다.

- `idle`: 실행 전 또는 리셋 후 대기
- `preparing`: 준비 카운트다운 실행 중
- `running`: 타이머 실행 중
- `paused`: 일시정지
- `completed`: 종료
- `editing`: 설정 편집 중

### 공통 조작

- 시작: 현재 선택한 타이머를 실행한다.
- 일시정지: 실행 중인 타이머를 멈춘다.
- 재개: 일시정지된 타이머를 이어서 실행한다.
- 리셋: 현재 타이머를 초기 상태로 되돌린다.
- 편집: 현재 타이머 설정을 수정한다.
- 저장: 편집 내용을 저장한다.
- 취소: 편집을 중단하고 이전 화면으로 돌아간다.

## 핵심 타이머 기능

### 1. 타이머

타이머는 하나의 화면에서 카운트다운과 카운트업을 토글로 전환한다.
카운트다운과 카운트업은 별도 최상위 메뉴나 별도 페이지로 노출하지 않는다.

- 경로: `/timer`
- 모드: `countdown`, `count_up`
- 기본 모드: `countdown`
- 모드 전환: 화면 우측 상단 토글 버튼
- 표시 형식: `HH:MM:SS`
- 설정 가능 범위: `00:00:00`부터 `23:59:59`
- 시간, 분, 초는 키보드 입력이 아니라 드래그 picker로 설정한다.
- 시작 전에는 시간, 분, 초를 수정할 수 있어야 한다.
- 실행 중에는 시간, 분, 초를 수정할 수 없어야 한다.
- 모드 토글은 타이머 상태와 무관하게 항상 접근 가능해야 한다.
- 실행 중 버튼은 `시작`에서 `일시정지`로 바뀐다.
- 일시정지 상태에서는 `재개`와 `리셋`을 제공한다.
- 종료 시 알림 큐가 활성화되어 있으면 종료 알림을 실행한다.
- 종료 시 휴대폰 네이티브 알림 배너를 표시할 수 있어야 한다.
- 동일 설정을 다시 실행할 수 있어야 한다.

#### 카운트다운 모드

카운트다운은 설정한 시간에서 `00:00:00`까지 감소한다.

- 종료 시 `00:00:00`에서 멈춘다.
- 종료 시 진동 또는 사운드 큐를 실행할 수 있어야 한다.

#### 카운트업 모드

카운트업은 `00:00:00`에서 시작해 설정한 종료 시간까지 증가한다.

- 항상 `00:00:00`에서 시작한다.
- 설정한 종료 시간에 도달하면 멈춘다.
- 종료 시 진동 또는 사운드 큐를 실행할 수 있어야 한다.

카운트업은 타이머의 모드일 뿐, 별도 라우트로 분리하지 않는다.

### 2. 스톱워치

스톱워치는 시간을 증가 방향으로 측정한다.

- 표시 형식: `MM:SS.CS`
- `CS`는 1/100초를 의미한다.
- 시작값: `00:00.00`
- 최대값: `99:59.99`
- 시작, 일시정지, 재개, 리셋이 가능해야 한다.
- 별도 프로그래밍 설정은 제공하지 않는다.
- 준비 카운트다운은 적용하지 않는다.
- 기본적으로 알림 큐는 적용하지 않는다.

### 3. 인터벌 타이밍

인터벌 타이밍은 여러 개의 운동 구간과 휴식 구간을 라운드 단위로 실행한다.

- 각 인터벌 세트는 운동 시간과 휴식 시간을 가진다.
- 운동 시간 표시 형식: `Fn MM:SS`
- 휴식 시간 표시 형식: `Cn MM:SS`
- `n`은 1부터 시작하는 인터벌 번호다.
- 하나의 인터벌 프리셋은 최대 9개의 인터벌 세트를 가질 수 있다.
- 하나의 인터벌 프리셋은 최대 99라운드까지 설정할 수 있다.
- 운동 시간과 휴식 시간은 `00:00`부터 `99:59`까지 설정할 수 있다.
- 실행 중 일시정지와 재개가 가능해야 한다.
- 동일 설정을 다시 빠르게 실행할 수 있어야 한다.
- 준비 카운트다운을 적용할 수 있어야 한다.

#### 인터벌 실행 순서

인터벌은 다음 순서로 실행한다.

1. 준비 카운트다운이 켜져 있으면 준비 카운트다운을 실행한다.
2. 1라운드의 `F1` 운동 시간을 실행한다.
3. 1라운드의 `C1` 휴식 시간을 실행한다.
4. 설정된 마지막 인터벌 세트까지 `Fn`, `Cn`을 순서대로 실행한다.
5. 다음 라운드가 있으면 2번부터 반복한다.
6. 마지막 라운드의 마지막 구간이 끝나면 종료한다.

휴식 시간이 `00:00`인 경우 해당 휴식 구간은 건너뛴다.

## 인터벌 프리셋

Tabata, FGB, EMOM은 별도의 타이머 엔진으로 구현하지 않는다. 모두 인터벌 타이밍의 기본 제공 프리셋으로 표현한다.

### 기본 제공 프리셋

기본 제공 프리셋은 앱 설치 시 제공되는 템플릿이다.

- 삭제할 수 없다.
- 바로 실행할 수 있어야 한다.
- 복제하여 사용자 프리셋으로 저장할 수 있어야 한다.
- 복제한 프리셋은 이름과 설정을 편집할 수 있어야 한다.

#### Tabata

- 운동 시간: 20초
- 휴식 시간: 10초
- 라운드: 8라운드

#### FGB 5R

- 운동 시간: 5분
- 휴식 시간: 1분
- 라운드: 5라운드

#### FGB 3R

- 운동 시간: 5분
- 휴식 시간: 1분
- 라운드: 3라운드

#### EMOM

EMOM은 휴식 시간이 `00:00`인 인터벌 프리셋으로 표현한다.

- 반복 시간: 사용자가 설정
- 휴식 시간: 0초
- 반복 횟수: 최대 99회
- 실행 중 현재 반복 번호와 남은 시간을 표시한다.

예시:

- 반복 시간: 30초
- 휴식 시간: 0초
- 반복 횟수: 3회
- 실행 표시:
  - `1 00:30`
  - `2 00:30`
  - `3 00:30`

### 사용자 프리셋

사용자는 직접 프리셋을 만들고 이름을 붙일 수 있어야 한다.

- 생성 가능
- 이름 변경 가능
- 설정 편집 가능
- 삭제 가능
- 복제 가능
- 바로 실행 가능
- 최근 사용한 설정을 프리셋으로 저장 가능

사용자 프리셋 이름 예시:

- `월요일 와드`
- `복싱 3분 1분`
- `버피 40/20`
- `스트렝스 휴식 타이머`

### 프리셋 데이터

프리셋은 최소한 다음 정보를 가진다.

- 이름
- 타이머 유형
- 라운드 수
- 인터벌 세트 목록
- 준비 카운트다운 설정
- 알림 큐 설정

## 알림 큐

알림 큐는 타이머 상태 변화를 사용자에게 알려주는 피드백 시스템이다.
기존 매뉴얼의 사운드 큐를 확장하여 사운드, 진동, 음원 파일, 직접 녹음 음성까지 포함한다.

### 적용 대상

- 타이머
- 인터벌 타이밍
- 기본 제공 인터벌 프리셋
- 사용자 프리셋

### 미적용 대상

- 스톱워치

### 알림 방식

사용자는 알림 방식을 선택할 수 있어야 한다.

- 없음
- 사운드
- 진동
- 사운드 + 진동

무음 모드이거나 소리를 낼 수 없는 환경에서는 진동으로 대체할 수 있어야 한다.

### 알림 이벤트

알림 큐는 다음 이벤트에 연결할 수 있어야 한다.

- 준비 카운트다운 시작
- 준비 카운트다운 중 지정 초 도달
- 운동 시작
- 휴식 시작
- 구간 종료
- 라운드 종료
- 마지막 라운드 진입
- 전체 종료

### 시작 전 알림

사용자는 시작 전 알림 시점을 선택할 수 있어야 한다.

- 없음
- 1초 전
- 3초 전
- 5초 전
- 10초 전

기본값은 3초 전으로 한다. 10초 전부터 계속 울리는 방식은 사용자가 길다고 느낄 수 있으므로 기본값으로 사용하지 않는다.

### 준비 카운트다운

준비 카운트다운은 타이머 본 실행 전에 실행되는 준비 단계다.

- 적용 대상:
  - 카운트다운
  - 카운트업
  - 인터벌 타이밍
  - 기본 제공 인터벌 프리셋
  - 사용자 프리셋
- 미적용 대상:
  - 스톱워치
- 기본값은 10초다.
- 사용자는 준비 카운트다운을 끌 수 있어야 한다.
- 사용자는 준비 카운트다운 길이를 조정할 수 있어야 한다.
- 준비 카운트다운의 알림 시점은 시작 전 알림 설정을 따른다.

### 기본 사운드

MVP는 기본 사운드를 제공해야 한다.

- 짧은 비프
- 긴 비프
- 종료음

알림 이벤트별 기본 사운드는 다음과 같다.

- 운동 시작: 짧은 비프
- 휴식 시작: 짧은 비프
- 라운드 종료: 짧은 비프
- 전체 종료: 긴 비프 또는 종료음

### 음원 파일 커스텀

사용자는 알림 이벤트에 사용할 음원 파일을 직접 지정할 수 있어야 한다.

- 기기 내 음원 파일 선택
- 이벤트별 음원 지정
- 지정한 음원 미리듣기
- 지정한 음원 삭제 또는 기본값 복원

MVP에서는 구조만 고려하고, 실제 음원 파일 선택 기능은 후속 단계로 구현해도 된다.

### 직접 녹음 음성 큐

사용자는 직접 녹음한 코치 목소리를 알림 큐로 사용할 수 있어야 한다.

예시:

- `준비`
- `3, 2, 1`
- `고`
- `쉬어`
- `마지막 라운드`
- `10초 남았어`
- `끝`

직접 녹음 음성 큐는 다음 기능을 제공한다.

- 앱에서 직접 녹음
- 녹음 파일 이름 지정
- 이벤트별 녹음 파일 지정
- 미리듣기
- 다시 녹음
- 삭제

이 기능은 제품 매력을 높이는 차별화 기능이다. MVP에서는 알림 큐 구조가 이 기능을 수용할 수 있도록 설계하고, 실제 녹음 UI는 후속 단계로 구현해도 된다.

## 밝기 및 표시 설정

기존 하드웨어 타이머는 5단계 밝기 조절을 제공한다.

tempo는 화면 미러링과 모바일 표시를 고려해 다음 중 하나를 제공해야 한다.

- 앱 내부 타이머 화면 밝기 5단계 조절
- 또는 기기 밝기와 별개인 타이머 화면 대비/밝기 테마 5단계 조절

하드웨어 디스플레이 연동은 현재 범위에서 제외한다.

## 모바일 제어 매핑

기존 리모컨 조작은 모바일 UI로 대체한다.

| 기존 리모컨 | tempo 모바일 적용 |
| --- | --- |
| Power | 앱 실행/화면 켜짐 상태로 대체 |
| Clock | 제공하지 않음 |
| 12Hour / 24Hour | 제공하지 않음 |
| Down | 타이머에서 카운트다운 모드 선택 |
| Up | 타이머에서 카운트업 모드 선택 |
| Stopwatch | 스톱워치 모드 선택 |
| Tabata | 기본 제공 인터벌 프리셋 선택 |
| FGB | 기본 제공 인터벌 프리셋 선택 |
| START | 시작 또는 재개 |
| STOP | 일시정지 |
| Reset | 현재 타이머 리셋 또는 편집 취소 |
| SET / EDIT | 현재 설정 또는 프리셋 편집 |
| OK | 설정 저장 |
| 좌/우 방향 버튼 | 시간, 분, 초 picker 선택으로 대체 |
| 숫자 0-9 | 타이머에서는 제공하지 않음. 인터벌 편집에서만 필요 시 숫자 입력으로 대체 |
| P0-P9 | 이름 있는 프리셋 목록으로 대체 |
| 10Sec | 준비 카운트다운 설정으로 대체 |
| Buzzer | 알림 큐 설정으로 대체 |
| Brightness | 밝기 또는 표시 테마 단계 변경 |
| Invalid buttons | 제공하지 않음 |

### 숫자 버튼과 프리셋 처리

기존 리모컨의 `0-9` 버튼은 두 역할을 가진다.

- 편집 중: 숫자 입력
- 일반 상태: `P0-P9` 인터벌 슬롯 선택

모바일에서는 이중 역할을 그대로 강제하지 않는다. 다음 UI로 분리한다.

- 타이머 화면: 시간, 분, 초 드래그 picker 제공
- 인터벌 편집 화면: 필요 시 숫자 입력 제공
- 프리셋 화면: 이름 있는 프리셋 목록 제공
- 프리셋 상세 화면: 실행, 편집, 이름 변경, 복제, 삭제 제공

기존 `P0-P9` 슬롯 개념은 매뉴얼 호환 참고사항으로만 둔다. tempo의 실제 UX는 이름 있는 프리셋 목록을 기준으로 한다.

## 표시 요구사항

### 모바일 기본 화면

- 현재 모드
- 남은 시간 또는 경과 시간
- 현재 라운드
- 전체 라운드
- 현재 구간: 준비, 운동, 휴식, 완료
- 알림 큐 상태
- 준비 카운트다운 상태
- 시작/일시정지/리셋 버튼

### 미러링 화면

모바일 화면을 TV나 외부 화면에 미러링할 수 있으므로, 타이머 실행 화면은 멀리서도 읽을 수 있어야 한다.

- 시간 숫자는 화면에서 가장 크게 표시한다.
- 현재 구간은 색상과 텍스트로 구분한다.
- 컨트롤 버튼이 숫자 표시를 방해하지 않아야 한다.
- 가로 화면에서 보기 좋은 레이아웃을 지원해야 한다.

## 구현 예시

### 타이머: 카운트다운 30분

- 모드: 카운트다운
- 시작 시간: `00:30:00`
- 시작 시 표시: `00:30:00`
- 종료 시 표시: `00:00:00`

### 타이머: 카운트업 30분

- 모드: 카운트업
- 종료 시간: `00:30:00`
- 시작 시 표시: `00:00:00`
- 종료 시 표시: `00:30:00`

### 인터벌 예시 1

- 이름: `복싱 3분 1분`
- 운동 시간: 3분
- 휴식 시간: 1분
- 라운드: 4라운드
- 실행 순서:
  - `F1 03:00`
  - `C1 01:00`
  - 위 순서를 4회 반복

### 인터벌 예시 2

원본 매뉴얼에는 이 예시에 일부 오탈자가 있으므로, 아래 명세를 기준으로 구현한다.

- 이름: `피라미드 인터벌`
- 인터벌 1: 운동 90초, 휴식 30초
- 인터벌 2: 운동 60초, 휴식 20초
- 인터벌 3: 운동 30초, 휴식 10초
- 라운드: 8라운드
- 실행 순서:
  - `F1 01:30`
  - `C1 00:30`
  - `F2 01:00`
  - `C2 00:20`
  - `F3 00:30`
  - `C3 00:10`
  - 위 순서를 8회 반복

### EMOM 예시

- 이름: `EMOM 10`
- 운동 시간: 1분
- 휴식 시간: 0초
- 라운드: 10라운드

## 검수 기준

- 실시간 시계 모드는 제공하지 않아야 한다.
- 모든 타이머 모드는 시작, 일시정지, 재개, 리셋 동작이 명확해야 한다.
- 타이머는 하나의 화면 우측 상단 토글 버튼으로 카운트다운과 카운트업을 전환해야 한다.
- 타이머는 `23:59:59`까지 지원해야 한다.
- 스톱워치는 `99:59.99`까지 표시할 수 있어야 한다.
- 인터벌 프리셋은 최대 9개 인터벌 세트와 최대 99라운드를 지원해야 한다.
- Tabata, FGB, EMOM은 별도 엔진이 아니라 기본 제공 인터벌 프리셋으로 동작해야 한다.
- 사용자는 직접 프리셋을 만들고 이름을 붙일 수 있어야 한다.
- 사용자는 프리셋을 편집, 복제, 삭제, 실행할 수 있어야 한다.
- 알림 큐는 사운드, 진동, 사운드 + 진동, 없음 모드를 지원해야 한다.
- 시작 전 알림 시점은 없음, 1초, 3초, 5초, 10초 중 선택할 수 있어야 한다.
- 알림 큐 구조는 음원 파일 커스텀과 직접 녹음 음성 큐를 수용할 수 있어야 한다.
- 모바일 UI는 리모컨 없이 모든 기능을 제어할 수 있어야 한다.
- 미러링 상황에서도 실행 중 시간과 상태를 멀리서 읽을 수 있어야 한다.
